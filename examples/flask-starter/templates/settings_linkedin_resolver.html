API keys

Paste your Apollo, Google Custom Search, and OpenAI keys here. The app uses them in three places: LinkedIn resolution (Supporters page), Supporter Scanner LinkedIn enrichment, and Auto-Prospecting.

What these keys power

Three features share the same keys, so configuring them here lights up everything at once:

  • LinkedIn resolver — turns an email into a LinkedIn URL so you can import a contact as a Target with one click.
  • Supporter Scanner enrichment — resolves Gmail/Calendar contacts so their connector-card badges line up across teammates.
  • Auto-Prospecting — uses gpt-4o-mini to expand titles + Apollo's people search to surface lookalikes at an account.

For LinkedIn resolution specifically we try two methods, in order:

  1. Apollo — fastest. One API call, returns the LinkedIn URL directly when Apollo knows the person. Hits 50–70% of business contacts.
  2. Google search + AI ranking — fallback. Searches the web for "{first name} {company} linkedin", then asks gpt-4o-mini to pick the right profile. Slower but cheap.

All three keys are optional. Leave any blank and we just skip that method. If neither method finds the URL, the row stays usable — you can paste a LinkedIn URL manually before importing.

Privacy: every lookup runs locally on your laptop. Names + emails go to Apollo and Google CSE per-call as needed. Draftboard's servers never see this data.

1. Apollo

Recommended — fastest method

Apollo's /people/match endpoint accepts an email + name and returns the LinkedIn URL when it knows the person. Setup time: ~2 minutes.

  1. Sign up at apollo.io (free tier exists).
  2. Open Settings → Integrations → API.
  3. Click Create New Key and copy the value.
  4. Paste it below and Save.

2. Google Custom Search

Fallback when Apollo misses

Search the web for the person's LinkedIn profile. Free for the first 100 lookups per day. Setup time: ~5 minutes.

  1. Open programmablesearchengine.google.com and click Add.
  2. Name it anything (e.g. "LinkedIn finder"). Toggle Search the entire web on. Click Create.
  3. On the resulting page, copy the Search engine ID (looks like a long alphanumeric string).
  4. Open Custom Search API in Google Cloud Console, click Enable.
  5. Go to Credentials → Create Credentials → API key. Copy the key.
  6. Paste both below and Save.
